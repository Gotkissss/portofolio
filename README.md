# Reflexión

## ¿A qué tipo de audiencia está dirigido este portafolio?

Este portafolio está pensado principalmente para startups o equipos de desarrollo pequeños que buscan a alguien que pueda aportar más que solo crear interfaces. Aunque mi enfoque principal es el desarrollo frontend, también me interesa mostrar conocimientos de backend, arquitectura de software y seguridad informática.

Por esa razón, en lugar de incluir únicamente una lista de tecnologías, preferí presentar proyectos reales y explicar cómo fueron desarrollados. De esta forma, las personas que revisen el portafolio pueden conocer tanto el resultado final como la forma en que trabajo y tomo decisiones durante el desarrollo.

El diseño también está orientado a ese tipo de audiencia. La estética inspirada en terminales y el estilo minimalista reflejan mi interés por la programación y la ciberseguridad, dos áreas que forman parte de mi perfil.

## ¿Qué tecnologías elegí usar y por qué?

### React + Vite

Utilicé React junto con Vite para desarrollar el portafolio. React es la herramienta con la que tengo más experiencia y me permite organizar mejor los componentes y la interfaz. Vite fue elegido porque facilita el desarrollo gracias a su rapidez y configuración sencilla.

### JavaScript Vanilla con ES Modules

En el proyecto **Series Tracker** decidí utilizar JavaScript Vanilla con módulos ES. Quería demostrar que puedo desarrollar aplicaciones sin depender completamente de frameworks y que entiendo los conceptos fundamentales que utilizan muchas herramientas modernas.

### Go

Para el backend de Series Tracker utilicé Go. Aunque el portafolio está enfocado principalmente en frontend, también quería mostrar experiencia trabajando del lado del servidor y ampliar mi perfil como desarrollador.

### GitHub Actions + GitHub Pages

Utilicé GitHub Actions y GitHub Pages para automatizar el despliegue del portafolio. Esto permite publicar cambios de manera sencilla y mantener un flujo de trabajo más ordenado.

## ¿Qué tecnología decidí no usar?

Decidí no utilizar frameworks de CSS como Bootstrap o Tailwind. Todos los estilos fueron escritos directamente dentro de los componentes utilizando CSS-in-JS.

Tomé esta decisión porque quería tener control total sobre la apariencia del portafolio y evitar que se sintiera como una plantilla prediseñada. Además, fue una buena oportunidad para demostrar mis conocimientos de CSS sin depender de librerías externas.

Reconozco que esta decisión hace que el código sea un poco más extenso. En un proyecto desarrollado en equipo probablemente utilizaría herramientas como Tailwind para trabajar más rápido, pero para este proyecto personal preferí enfocarme en la personalización.

## ¿Dónde me arriesgué y dónde la jugué seguro?

La parte más segura fue la elección del stack. React, Vite y JavaScript son tecnologías con las que ya he trabajado bastante, así que sabía que podía construir una aplicación estable sin mayores inconvenientes.

Donde sí decidí experimentar fue en el diseño. Implementé un cursor personalizado, animaciones cuando los proyectos aparecen en pantalla y una estética inspirada en terminales. Estas decisiones le dieron más personalidad al portafolio, aunque también implicaban el riesgo de alejarse de un diseño más tradicional.

Además, decidí no incluir una sección de contacto extensa ni una fotografía personal. Preferí que la atención estuviera centrada en los proyectos y en el código.

## Si tuviera otra semana, ¿qué mejoraría?

Lo primero que haría sería agregar un proyecto de backend desarrollado completamente en Go para mostrar de manera más clara esa parte de mis habilidades. Actualmente la mayoría de los proyectos están enfocados en frontend, por lo que un proyecto adicional ayudaría a presentar un perfil más completo.

También dedicaría más tiempo a mejorar la experiencia en dispositivos móviles. El portafolio funciona correctamente en teléfonos y tablets, pero todavía hay detalles de espaciado, tamaños de texto y distribución de elementos que podrían optimizarse para ofrecer una mejor experiencia.
